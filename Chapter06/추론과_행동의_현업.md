## 📝 Chapter 6: 추론과 행동의 협업 (ReAct)

### 1. ReAct 기반 체계의 이해

ReAct(Reasoning + Acting)는 LLM이 복잡한 과제를 해결할 때 '추론(Reasoning)'과 '행동(Acting)'을 결합하여 최적의 경로를 생성하는 프레임워크입니다. 단순히 정답을 내놓는 것이 아니라, 환경을 관찰하고 사고하며 단계적으로 정답에 다가갑니다.

#### **꽃가게 자동 가격 설정 시나리오**

상황: 날씨 및 시장 상황에 따라 유동적인 꽃 가격 결정이 필요한 경우

1. **사고 (Thought)**: "현재 날씨와 시장 상황을 고려하여 오늘 장미의 적정 판매가를 결정해야겠다."
2. **행동 (Action)**: 검색 도구를 호출하여 오늘의 꽃 도매 원가와 날씨 정보를 확인.
3. **관찰 (Observation)**: "원가가 10% 상승했고, 오늘 비가 올 확률이 80%구나."
4. **사고 (Thought)**: "원가 상승분과 수요 감소 가능성을 고려해 마진율을 15%로 조정하자."
5. **행동 (Action)**: 최종 판매 가격을 계산하여 시스템에 업데이트.

### 2. LangChain 에이전트 구현의 4대 핵심 요소

LangChain은 ReAct 체계를 패키징하여 개발자가 에이전트를 쉽게 구현할 수 있도록 돕습니다. 에이전트 구동을 위해 다음 네 가지 요소가 반드시 필요합니다.

| 구성 요소 | 역할 | 비유 |
| --- | --- | --- |
| **LLM** | 논리적 엔진 제공, 예측 생성 및 입력 처리 | **에이전트의 '뇌'** |
| **프롬프트** | 모델의 페르소나 설정 및 추론 기반 체계 형성 | **업무 매뉴얼** |
| **외부 도구 (Tools)** | 검색 엔진, DB 조회, API 호출 등 실무 수행 | **에이전트의 '도구함'** |
| **에이전트 실행기** | 루프를 관리하며 적절한 도구를 호출하고 순서 제어 | **에이전트의 '운영체제'** |

---

### 3. 에이전트의 작동 원리: 관찰-추론-행동

LangChain 에이전트는 고도의 자율성을 바탕으로 작업을 수행하며, 프롬프트를 통해 그 성격과 맥락이 결정됩니다.

* **자율성 부여**: LLM은 단순히 텍스트를 생성하는 수준을 넘어, 주어진 도구 중 어떤 것을 언제 사용할지 스스로 판단합니다.
* **맥락 유지**: 프롬프트에 포함된 작업의 배경 지식은 에이전트가 더 정교한 '사고'를 할 수 있도록 돕습니다.
* **피드백 루프**: 관찰(Observation) 결과를 다시 사고(Thought)의 입력값으로 사용하여 에러를 수정하거나 다음 단계를 계획합니다.

### 4. 핵심 비교: 정적 워크플로우 vs ReAct 에이전트

| 구분 | 일반적인 프로그래밍 (Chain) | ReAct 에이전트 (Agent) |
| --- | --- | --- |
| **경로 결정** | 개발자가 미리 정의한 순서대로 실행 | LLM이 상황에 맞춰 실시간으로 경로 생성 |
| **유연성** | 예외 상황 대응이 어려움 | 관찰 결과에 따라 행동을 유연하게 수정 |
| **적합한 작업** | 단계가 명확하고 고정된 작업 | 복합적인 판단이나 외부 데이터 확인이 필요한 작업 |

---

### 5. 요약 및 결론

* **ReAct의 가치**: "생각하고 행동하라"는 인간의 문제 해결 방식을 AI에 이식하여, 예측 불가능한 외부 환경(날씨, 시장가 등)에 대응할 수 있게 합니다.
* **LangChain의 역할**: 복잡한 ReAct 로직을 추상화하여, 개발자가 LLM과 도구만 연결하면 스스로 판단하고 움직이는 에이전트를 만들 수 있게 지원합니다.
* **미래 전망**: 단순한 챗봇을 넘어, 기업의 실시간 비즈니스 로직(가격 정책, 재고 관리 등)을 자동화하는 지능형 운영 엔진으로 진화할 것입니다.
